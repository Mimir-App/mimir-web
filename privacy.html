Politica de Privacidad

Ultima actualizacion: 24 de marzo de 2025

1. Introduccion

Mimir es una aplicacion de escritorio de codigo abierto para la gestion inteligente del tiempo. Esta politica de privacidad explica como se recopilan, utilizan y protegen tus datos al usar Mimir.

Principio fundamental: Mimir esta diseñado para que tus datos permanezcan en tu ordenador. No operamos servidores que recopilen informacion de los usuarios.

2. Datos que recopila Mimir

2.1. Datos de actividad (captura automatica)

Cuando el servicio de captura esta activo, Mimir registra localmente:

  • Nombre de la ventana activa y aplicacion
  • Proyecto git y rama del directorio de trabajo
  • Estado de inactividad del sistema
  • Estado de audio y reuniones (deteccion de actividad, no contenido)

Estos datos se almacenan exclusivamente en una base de datos SQLite en tu ordenador (~/.config/mimir/daemon.db). Nunca se transmiten a servidores externos de Mimir.

2.2. Datos de configuracion

Mimir almacena localmente tus preferencias, credenciales de integracion y configuracion de la aplicacion. Los tokens sensibles se guardan en el keyring del sistema operativo.

3. Integraciones con terceros

Mimir se conecta con servicios externos solo cuando tu lo configuras explicitamente. Cada integracion implica comunicacion directa entre tu ordenador y el servicio, sin intermediarios:

3.1. Google Calendar

  • Datos accedidos: eventos del calendario (solo lectura)
  • Proposito: detectar reuniones para enriquecer los bloques de actividad
  • Autorizacion: OAuth 2.0 con scope calendar.readonly
  • Tokens: almacenados localmente en ~/.config/mimir/google_tokens.json
  • Puedes revocar el acceso en cualquier momento desde la aplicacion o desde tu cuenta de Google

3.2. Odoo

  • Datos enviados: bloques de horas confirmados (descripcion, proyecto, tarea, duracion)
  • Datos recibidos: proyectos, tareas, partes de horas, estado de fichaje
  • Comunicacion: directa con tu instancia de Odoo mediante XMLRPC o REST API

3.3. GitLab y GitHub

  • Datos recibidos: issues, merge requests/pull requests, comentarios, labels, notificaciones
  • Comunicacion: directa con tu instancia de GitLab o con la API de GitHub
  • Autenticacion: Personal Access Token o OAuth Device Flow (GitHub)

3.4. Proveedores de IA

  • Datos enviados: contexto del bloque de actividad (aplicaciones usadas, proyecto, rama) para generar descripciones
  • Proveedores soportados: Google Gemini, Anthropic Claude, OpenAI
  • Nota: los datos enviados dependen del proveedor que configures. Consulta la politica de privacidad de cada proveedor

4. Datos que NO recopilamos

  • No recopilamos datos de uso ni telemetria
  • No rastreamos tu actividad fuera de la aplicacion
  • No almacenamos datos en servidores propios
  • No compartimos datos con terceros (excepto las integraciones que tu configuras)
  • No capturamos contenido de pantalla, pulsaciones de teclado ni contenido de archivos

5. Almacenamiento y seguridad

  • Todos los datos se almacenan localmente en tu ordenador
  • La base de datos SQLite se encuentra en ~/.config/mimir/
  • Los tokens sensibles se almacenan en el keyring del sistema operativo
  • La comunicacion con servicios externos utiliza HTTPS
  • Puedes eliminar todos tus datos borrando el directorio ~/.config/mimir/

6. Retencion de datos

Mimir aplica politicas de retencion configurables:

  • Señales de actividad: 180 dias por defecto
  • Bloques: 365 dias por defecto
  • Puedes modificar estos valores en la configuracion
  • Puedes eliminar datos manualmente en cualquier momento

7. Tus derechos

Al ser una aplicacion local, tienes control total sobre tus datos:

  • Acceso: todos los datos estan en tu ordenador, accesibles directamente
  • Eliminacion: borra ~/.config/mimir/ para eliminar todo
  • Portabilidad: la base de datos SQLite es un formato estandar y abierto
  • Revocacion: desconecta cualquier integracion desde la configuracion de Mimir

8. Cambios en esta politica

Publicaremos cualquier cambio en esta pagina. Al ser una aplicacion de escritorio, las actualizaciones de la politica se reflejan con nuevas versiones de la aplicacion.
